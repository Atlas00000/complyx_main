# Framework, Scoring & Implementation Plan

---

## Part 1: Framework & Scoring System

### Question Structure

| Assessment Type | Questions | Duration | Coverage |
|-----------------|-----------|----------|----------|
| **Quick** | 5–8 | ~5 min | 1–2 per pillar |
| **Micro** | 3–5 | ~3 min | Single pillar |
| **Full** | 18–20 | ~15 min | All pillars |

### Pillars (IFRS S1/S2)

1. **Governance** – Oversight, roles, processes
2. **Strategy** – Approach, integration, targets
3. **Risk** – Identification, assessment, management
4. **Metrics & Targets** – KPIs, measurement, disclosure

### Question Types & Mix

| Type | % | Scoring |
|-----|---|---------|
| Multiple choice | 50% | 0/25/50/75/100 by option |
| Yes/No | 25% | Yes=100, No=0 |
| Scale (1–5) | 15% | 1→0%, 2→25%, 3→50%, 4→75%, 5→100% |
| Open text | 10% | Context only (no score) |

### Scoring

- **Category score** = average of question scores in that pillar
- **Overall score** = weighted average of pillars (25% each)
- **Gap** = question or pillar < 50%
- **Readiness bands:** 0–19%, 20–39%, 40–59%, 60–79%, 80–100%

### Selectable Assessment Flow

```
User triggers assessment
    → AI: "Which type? Quick, Micro, or Full?"
    → User selects (buttons or text)
    → If Micro: "Which topic?" → Governance/Strategy/Risk/Metrics
    → AI starts appropriate question set
```

## Part 1.5: Question Bank

### Pillar 1: Governance

**G1. Board oversight** (MC)  
*"How does your board oversee sustainability and climate-related matters?"*

- A) Dedicated sustainability committee with regular reporting (100)
- B) Part of existing committee (e.g. audit/risk) with defined responsibilities (75)
- C) Ad hoc board discussions (50)
- D) No formal board oversight (0)

---

**G2. Management responsibility** (Yes/No)  
*"Does your organization have a designated executive or team responsible for sustainability and climate disclosures?"*

- Yes (100) / No (0)

---

**G3. Governance maturity** (Scale 1–5)  
*"How formalized is your governance structure for sustainability reporting?"*

1 = Ad hoc, no formal structure  
2 = Informal, some roles identified  
3 = Documented roles and responsibilities  
4 = Formal policies and regular reporting  
5 = Integrated into board and management processes  

*Scale mapping: 1→0%, 2→25%, 3→50%, 4→75%, 5→100%*

---

**G4. Competence and training** (MC)  
*"How does your organization ensure competence in sustainability and climate reporting?"*

- A) Formal training programs for relevant staff (100)
- B) Access to external expertise when needed (75)
- C) Informal knowledge sharing (50)
- D) No specific measures (0)

---

**G5. Disclosure controls** (Yes/No)  
*"Do you have internal controls or processes to ensure the accuracy of sustainability disclosures?"*

- Yes (100) / No (0)

## Part 2: Weekly Implementation Phases

---

## Phase 1: Foundation (Weeks 1–2)

### Week 1: Data Model & Question Bank

| Day | Task |
|-----|------|
| 1 | Extend Prisma: `AssessmentSession`, `AssessmentType`, link to `User`/`Session` |
| 2 | API: `POST /assessment/start`, `POST /assessment/answer`, `GET /assessment/status` |
| 3 | Seed full question bank (18–20 questions) with pillar, type, options, scores |
| 4 | Assessment service: start session, submit answer, get next question |
| 5 | Scoring logic: category and overall scores, gap detection |

**Deliverables:** Schema, APIs, question bank, scoring service.

---

### Week 2: Assessment Type Selection & Basic Flow

| Day | Task |
|-----|------|
| 1 | Define question subsets: Quick (5–8), Micro (3–5 per topic), Full (18–20) |
| 2 | API: accept `type` and `topic` on start; return correct question set |
| 3 | Chat: detect "start assessment" → enter assessment mode |
| 4 | Chat: AI asks for type; render Quick/Micro/Full buttons |
| 5 | Chat: if Micro, ask topic; render Governance/Strategy/Risk/Metrics buttons |

**Deliverables:** Type selection in chat, correct question set per type.

---

## Phase 2: Chat Integration (Weeks 3–4)

### Week 3: Text-Based Assessment Flow

| Day | Task |
|-----|------|
| 1 | System prompt for assessment mode (type-aware) |
| 2 | AI sends Q1 → user replies → store answer → AI sends Q2 |
| 3 | Progress: "Question 3 of 8" (Quick) / "Question 2 of 5" (Micro) / "Question 10 of 18" (Full) |
| 4 | Completion: compute scores, gaps, readiness band; AI sends summary |
| 5 | Exit assessment mode; persist results |

**Deliverables:** End-to-end text-based assessment for all types.

---

### Week 4: Polish & Edge Cases

| Day | Task |
|-----|------|
| 1 | Skip / "I don't know" handling |
| 2 | Mid-assessment type switch (confirm, save, restart) |
| 3 | Resume partial assessment |
| 4 | Progress indicator in chat UI |
| 5 | Suggested prompts: "Start assessment", "Quick check", etc. |

**Deliverables:** Robust flow, UX polish.

---

## Phase 3: Rich Blocks & Scoring (Weeks 5–6)

### Week 5: Rich Message Blocks

| Day | Task |
|-----|------|
| 1 | Message schema: `type: 'assessment_question'`, payload for MC/YesNo/Scale |
| 2 | Server: return structured question blocks for applicable questions |
| 3 | Client: `AssessmentQuestionBubble` (MC, Yes/No, Scale) |
| 4 | Client: submit selection → store answer → next question |
| 5 | Mixed flow: text questions + rich blocks where applicable |

**Deliverables:** Interactive question blocks in chat.

---

### Week 6: Scoring Integration & Gaps

| Day | Task |
|-----|------|
| 1 | Map MC options to 0/25/50/75/100 |
| 2 | Map Scale 1–5 to 0–100% |
| 3 | Category scores and overall weighted score |
| 4 | Gap identification (questions < 50%) |
| 5 | Readiness band and summary message in completion |

**Deliverables:** Full scoring and gap reporting.

---

## Phase 4: Context & Advanced (Weeks 7–8)

### Week 7: Assessment Context in Chat

| Day | Task |
|-----|------|
| 1 | Assessment summary service (scores, gaps, key answers) |
| 2 | Inject summary into chat context/system prompt |
| 3 | AI uses assessment data for personalized answers |
| 4 | Handle no assessment / partial assessment |
| 5 | UI hint: "Using your assessment to personalize responses" |

**Deliverables:** Chat aware of assessment results.

---

### Week 8: Contextual Questions & Wrap-Up

| Day | Task |
|-----|------|
| 1 | Detect assessment-relevant info in normal chat |
| 2 | Map to framework; store as contextual answers |
| 3 | Update scores when new info is captured |
| 4 | Tune when to ask vs. answer |
| 5 | Testing, docs, final polish |

**Deliverables:** Contextual data capture and integration.
